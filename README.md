# 淘宝用户行为数据分析项目 (Taobao User Behavior Analysis)

本项目基于阿里巴巴提供的淘宝用户行为数据集（包含约 1 亿条记录），利用 **Python** 和 **DuckDB** 进行了大规模数据处理与探索性数据分析（EDA）。通过对用户从浏览到购买的全链路分析，挖掘用户活跃规律，构建业务转化漏斗。

## 1\. 项目背景

在电商领域，理解用户行为是提升转化率和优化用户体验的关键。本项目旨在解决以下问题：

  * 每日流量（PV/UV）的波动趋势如何？
  * 用户活跃度随小时段变化的规律？
  * 从点击、收藏、加购到购买的转化效率如何？
  * 如何从亿级数据中高效筛选出“活跃用户”？

## 2\. 数据集说明

  * **数据来源**：UserBehavior.csv（阿里巴巴官方数据集）
  * **数据量**：100,095,182 条记录
  * **核心维度**：
      * `uid`: 用户ID
      * `pid`: 商品ID
      * `cat_id`: 商品类目ID
      * `behavior`: 行为类型（0:浏览/PV, 1:收藏/FAV, 2:加购/CART, 3:购买/BUY）
      * `timestamp`: 时间戳（UTC格式）

## 3\. 技术栈

  * **数据处理**: [DuckDB](https://duckdb.org/) (针对大规模 CSV 的列式分析引擎)
  * **语言**: Python 3.x
  * **数据分析**: Pandas, NumPy
  * **可视化**: Matplotlib, Seaborn, Plotly (交互式图表)

## 4\. 核心分析流程

### A. 数据预处理

由于数据量巨大，项目使用了 DuckDB 的流式读取功能。

1.  **时区转换**：将原始 UTC 时间戳转换为北京时间（+8小时）。
2.  **特征提取**：从时间字段中分离出年、月、日、小时。
3.  **数据清洗**：剔除了重复项，并筛选了 2017-11-25 至 2017-12-03 这一完整的分析窗口。

### B. 关键指标计算

  * **PV (Page Views)**: 统计所有行为为 `0` 的记录。
  * **UV (Unique Visitors)**: 每日独立访问的用户数。
  * **活跃用户定义**: 满足 `每日 PV > 3` 或 `产生过收藏/加购/购买行为` 的用户。

### C. 漏斗模型

通过统计四种行为的分布情况，计算各环节的留存率和跳出率。

## 6\. 分析发现

  * **时间维度**：用户活跃度通常在晚上 20:00 - 22:00 达到巅峰，凌晨 02:00 - 05:00 为波谷。
  * **转化分析**：购买行为（BUY）相对于浏览（PV）的整体转化率较低，收藏和加购环节存在明显的用户流失，是运营优化的重点。
